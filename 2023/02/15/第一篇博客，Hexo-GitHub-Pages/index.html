第一篇博客，Hexo+GitHub Pages

发表于 2023-02-15

Hexo建立GitHub Pages

Github配置

  1. 新建repo，命名为username.github.io(比如hebe-tian.github.io)，点击创建
    目前免费用户只能创建公开仓库来使用GitHubPages
  2. 创建仓库之后会提示仓库中没有内容，作为一个码农，这必然是不能接受的一件事情，所以我们需要新建一个文件，并且在中间郑重的输入Hello World，然后把文件命名为index.html并保存
  3. 最后点击Settings-Pages，已经会提示我们个人网站的跳转链接，点击就可以跳转到新页面（如果提示了404，可以等两分钟再刷新）

2和3的操作是为了确认GitHubPages可以使用，并不会真的使用index.html这个页面

环境配置

  1. 安装，具体安装方式可以在juejin、zhihu、jianshu等平台搜索，有条件的同学可以Google
    • git
    • node，npm
  2. 使用命令行，输入npm install -g hexo-cli，如果提示没有权限则再次输入sudo npm install -g hexo-cli，会提示输入密码
  3. 输入hexo -v查看是否安装成功，命令行返回hexo信息就表示hexo已安装
  4. 使用在命令行输入npm install hexo-deployer-git --save，如果提示没有权限则再次输入sudo npm install hexo-deployer-git –-save（这个步骤也可以在后面执行）

Hexo本地

  1. 创建一个文件夹，命名为username.github.io
  2. 使用命令行工具，进入这个文件夹，执行hexo init对文件夹进行hexo初始化，成功后会提示”Start blogging with Hexo!”
  3. 测试hexo是否可用
    1. 在命令行输入hexo generate或者hexo g用来渲染页面
    2. 在命令行输入hero server或者hexo s用来打开本地服务器，命令行会提示可以访问http://localhost:4000
    3. 在浏览器访问http://localhost:4000 就可以看到初始化的博客页面，使用Control+C可以关闭服务
  4. 打开_config.yml文件（可以使用文本编辑或者编辑器），修改其中的信息，其中比较重要的：title、author、language（zh-CN）、timezone（Asia/Shanghai）并保存
  5. 此时重新打开命令行，先输入hexo clean清理缓存，然后再执行3中的操作，这个时候博客展示的内容就变成我们设置的内容

Hexo与GitHub Pages

目前博客只能在本地访问，如果需要关联到github上，我们需要另外在_config.yml文件中配置

  1. 在github对应的仓库中使用ssh形式clone，得到对应的SSH key
  2. 打开_config.yml文件，找到deploy对它进行配置
  3. 编辑config文件 需要注意所有:之后都要带一个空格，否则不会生效
  4. 部署到GitHub Pages
    1. 打开命令行，先输入hexo clean清理缓存
    2. 在命令行输入hexo generate或者hexo g用来渲染页面
    3. 在命令行输入hexo server或者hexo s用来打开本地服务器，命令行会提示可以访问http://localhost:4000
    4. 调试完成之后使用hexo deploy或者hexo d用来把博客部署到git的服务器上
      这一步需要注意，如果之前没有执行过npm install hexo-deployer-git --save，需要在hexo d之前执行
    5. 访问地址username.github.io，可以看到托管在git上的博客（第一次没有请求到的话，等两分钟再请求就能看到了）

发表新博客

打开命令行，执行hexo new “blogname”，可以在source/_post中看到新建的blogname.md的文件，修改title并在正文内写入内容随后重新执行部署命令就可以把新博客写入到GitHub Pages里
